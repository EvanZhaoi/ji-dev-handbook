# 05. Memo 模块开发步骤

## 1. 先定义数据模型

核心表：

- memo 主表
- memo 分组表
- memo 相关人表
- memo 用户收藏表
- memo 用户置顶表

置顶和收藏必须按用户存储，不能写在 memo 主表上。否则一个共享 Memo 被 A 置顶后，B 也会看到置顶。

## 2. 做最小列表

先完成：

- 查询列表
- 新建 Memo
- 选择 Memo
- 编辑标题和内容
- 保存

列表排序：

```text
当前用户置顶 DESC
memo.update_time DESC
memo.id DESC
```

## 3. 做编辑器

推荐使用 Milkdown Crepe：

- 默认可视化编辑
- 可切换 Markdown 源码
- 支持图片
- 支持表格、代码块、公式
- Mermaid 图形用 `@milkdown/plugin-diagram` + `mermaid`

注意：

- 编辑器初始化时可能触发一次内容更新，不应标记为未保存。
- 只有内容真正变化时才显示未保存。
- 切换 Memo、打开设置、打开通知前要自动保存或保留草稿。

## 4. 做相关人

相关人选择器建议做成远程搜索组件：

- 输入姓名、用户名、工号、邮箱
- 调后端成员搜索接口
- 选择后展示为可换行 tag
- 每个相关人可设置 `view` 或 `edit`

权限：

| 操作 | owner | edit | view |
|------|-------|------|------|
| 查看 | 是 | 是 | 是 |
| 编辑标题/正文/状态 | 是 | 是 | 否 |
| 修改分组 | 是 | 否 | 否 |
| 修改相关人 | 是 | 否 | 否 |
| 删除 | 是 | 否 | 否 |
| 收藏 | 是 | 是 | 是 |

## 5. 做列表性能

Memo 数量变多后，不要一次渲染所有 DOM。

使用：

- 服务端分页
- 滚动到底部加载下一页
- TanStack Virtual 虚拟列表

## 6. 做草稿保护

必须处理：

- 点击其他 Memo
- 点击设置
- 点击通知
- 系统通知跳转
- 切换编辑模式
- 关闭窗口

保存失败时：

- 不能阻止用户永远卡住。
- 应保留本地恢复草稿。
- 跳转可以继续，但再次打开该 Memo 时提示恢复。
