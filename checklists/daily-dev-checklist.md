# Daily Development Checklist

每天开始开发前：

- [ ] 拉取最新代码
- [ ] 确认 Node / Java / Rust / MySQL 环境正常
- [ ] 启动后端
- [ ] 启动桌面端
- [ ] 检查当前分支状态
- [ ] 明确今天只做一个主要功能

开发过程中：

- [ ] 优先复用开源组件
- [ ] 优先复用已有基础 UI 组件
- [ ] 不直接在组件里拼 API URL
- [ ] 不在业务组件里写复杂 SQL 或权限判断
- [ ] 需要保存用户状态时明确是本地状态还是服务端状态

提交前：

- [ ] 前端 `npm run build`
- [ ] 后端 `mvn test`
- [ ] `git diff --check`
- [ ] 手动测试主要流程
- [ ] 更新文档
